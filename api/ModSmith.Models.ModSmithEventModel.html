Remarks

Localization

events.json must contain localization keys for the event:

[ID].title       // The title of the event.

// The body text of the event when it first appears.
[ID].pages.INITIAL.description

// For each `EventOption` the event may offer, keys must appear of the form:
[OPTION_KEY].title  // The title of the option.
[OPTION_KEY].description  // The body text of the option, describing its effects.

LayoutType

The layout type of the event, which determines the core display behavior for the event. Almost all events use the Default layout.

Default layouts show a background image (See InitialPortraitPath) and offer the player a choice of options. Options can either end the event or continue it with a new set of options.

Combat layouts show a combat room and offer the player a choice of options. Note that this layout type is not required if an event may _trigger_ combat, only if it _begins_ in a combat room. Any event type can transition into combat using EnterCombatWithoutExitingEvent.

Custom layouts are an advanced option which provide their own Godot scene and are responsible for all their own logic and gameplay. When using a custom layout, you must supply your scene with LayoutScenePath.

Ancient layouts should not be used directly. Instead, subclass ModSmithAncientEventModel. 

public override EventLayoutType LayoutType { get; }

GenerateInitialOptions()

Generate the initial set of options the player may choose from when the event first appears.

Events act as a state machine, where each EventOption may either end the event with SetEventFinished or transition to the next state with SetEventState.
